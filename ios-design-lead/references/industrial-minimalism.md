<overview>
Industrial Minimalism blends the precision of Dieter Rams' Braun with the playful professionalism of Teenage Engineering. It's design that feels machined, intentional, and alive — professional tools with a human soul.

**Core tension:** Precision geometry + Playful spirit = Objects that feel like instruments
</overview>

<teenage_engineering>
## Teenage Engineering Design Philosophy

**Founded:** 2007, Stockholm, Sweden
**Founder:** Jesper Kouthoofd
**Signature product:** OP-1 synthesizer (in MoMA permanent collection)

<principle name="Playful Professionalism">
**Concept:** Professional capability disguised as approachable tools.

The OP-1 looks like a toy but is used by professional musicians worldwide. This deliberate tension:
- Lowers intimidation barrier
- Invites exploration
- Surprises with depth

**For iOS:** Make professional features feel accessible. Complexity should be discoverable, not overwhelming.
</principle>

<principle name="Limitation as Liberation">
**Concept:** Deliberate constraints inspire creativity.

The OP-1 has only 4 encoders, yet controls hundreds of parameters. Constraints force:
- Clear hierarchies
- Thoughtful organization
- User creativity

**For iOS:** Limit options per screen. 3-5 actions maximum. Depth through navigation, not density.
</principle>

<principle name="Color as Function">
**Concept:** Color maps to meaning, not decoration.

OP-1 encoders: Blue, Green, White, Orange — each controls different parameter type. Users learn the vocabulary.

**For iOS:**
- Single accent color per screen
- Color = function mapping
- Monochromatic base (grays, blacks, whites)
- Accent highlights interactive elements
</principle>

<principle name="Industrial Precision">
**Concept:** CNC-machined feel, precision geometry.

OP-1 housing: Thick aluminum, debossed logos, chamfered edges, subtle shadows that make it "float."

**For iOS:**
- Sharp edges with friendly corner radii (8-12pt)
- Subtle shadows for depth
- Elements that appear to lift off the surface
- Grid-locked precision in layout
</principle>

<visual_signatures>
**TE Visual Language:**

| Element | Execution |
|---------|-----------|
| Corners | Friendly rounded, but sharp overall form |
| Shadows | Subtle, creates "float" effect |
| Colors | Monochromatic + strategic orange accents |
| Typography | Clean sans-serif, generous spacing |
| Icons | Geometric, simple, functional |
| Layout | Grid-locked, asymmetric but balanced |
| Animation | Playful physics, spring-based |

**Orange as signature:**
- `#FF6B00` (light mode)
- `#FF8C42` (dark mode, slightly warmer)
- Use for primary CTA, active states, key affordances
</visual_signatures>
</teenage_engineering>

<braun_rams>
## Dieter Rams / Braun Design Philosophy

**Career:** Chief Designer at Braun 1961-1995
**Products:** SK4 "Snow White's Coffin," T3 Transistor Radio, countless others
**Influence:** Apple's design language, modern minimalism

<the_ten_principles>
## The 10 Principles of Good Design

<principle num="1" name="Good design is innovative">
Innovation doesn't mean novelty. It means finding better solutions.

**For iOS:** Use proven patterns. Innovate only when it genuinely improves experience.
</principle>

<principle num="2" name="Good design makes a product useful">
A product is bought to be used. Design emphasizes usefulness, ignores anything that detracts.

**For iOS:** Every element must serve function. Decoration = noise.
</principle>

<principle num="3" name="Good design is aesthetic">
Well-executed products are beautiful. Their aesthetic quality affects daily experience.

**For iOS:** Beauty through clarity, not ornamentation. Clean = beautiful.
</principle>

<principle num="4" name="Good design makes a product understandable">
At best, self-explanatory. No manual needed.

**For iOS:** Labels should be unnecessary. Affordances should be obvious. Actions should be clear.
</principle>

<principle num="5" name="Good design is honest">
No manipulation. No promises that can't be kept. No making things appear more powerful than they are.

**For iOS:** No dark patterns. No fake urgency. Buttons do exactly what they say.
</principle>

<principle num="6" name="Good design is unobtrusive">
Products are tools. Neutral, restrained. Room for self-expression.

**For iOS:** UI fades, content shines. The interface should be invisible during flow state.
</principle>

<principle num="7" name="Good design is long-lasting">
Avoids being fashionable, never appears antiquated.

**For iOS:** No trendy effects (glassmorphism, extreme gradients) unless functional. Flat, clean, timeless.
</principle>

<principle num="8" name="Good design is thorough down to the last detail">
Nothing arbitrary. Care and accuracy show respect for the user.

**For iOS:** Every pixel considered. Every animation timed. Every haptic synchronized.
</principle>

<principle num="9" name="Good design is environmentally friendly">
Conserves resources. Minimizes pollution. No waste.

**For iOS:** Efficient code. No unnecessary animations. Respect battery. Performance = sustainability.
</principle>

<principle num="10" name="Good design is as little design as possible">
Less, but better. Back to purity. Back to simplicity.

**For iOS:** "What can I remove?" Remove until only essence remains. Then remove one more thing.
</principle>
</the_ten_principles>

<visual_signatures>
**Braun Visual Language:**

| Element | Execution |
|---------|-----------|
| Form | Geometric, rectangular, honest materials |
| Color | Black, white, silver. Occasional red accent |
| Controls | Clear affordances, logical grouping |
| Typography | Functional, never decorative |
| Space | Generous, breathing room |
| Details | Consistent radii, aligned grids, harmonious proportions |
</visual_signatures>

<decision_filter>
**When uncertain, ask:**

"What would Dieter remove?"

If an element doesn't clarify function or improve experience, remove it.
</decision_filter>
</braun_rams>

<synthesis>
## Synthesized iOS Design Language

Combining Teenage Engineering and Braun for modern iOS:

<principle name="Industrial Precision, Playful Spirit">
**Visual:** CNC-machined feel — precision geometry, subtle shadows, monochromatic base
**Interaction:** Playful physics — springs, bounces, haptic feedback that surprises

Objects feel machined but alive.
</principle>

<principle name="Constraint as Clarity">
**Rams:** As little design as possible
**TE:** Limitation as liberation

Together: 3-5 actions per screen. Depth through navigation. Forced clarity.
</principle>

<principle name="Monochromatic + Accent">
**Rams:** Black, white, silver. Occasional red.
**TE:** Cool grays + strategic orange.

Together: Monochromatic base. Single accent color (orange recommended). Color = function.
</principle>

<principle name="Honest Interactions">
**Rams:** No manipulation, no false promises
**TE:** Professional capability, approachable presentation

Together: Buttons do what they say. Complex features are discoverable. Never lie to users.
</principle>

<principle name="Tactile Reality">
**Rams:** Physical products that feel substantial
**TE:** Industrial materials with playful interaction

Together: Haptics confirm actions. Springs give weight. Digital feels physical.
</principle>
</synthesis>

<color_system>
## Industrial Minimal Color System

**Base Palette:**
```
Light Mode:
- Background: #FAFAFA (off-white)
- Surface: #FFFFFF
- Primary text: #1A1A1A
- Secondary text: #6B6B6B
- Tertiary text: #9E9E9E
- Border: #E5E5E5

Dark Mode:
- Background: #0A0A0A (near-black)
- Surface: #1A1A1A
- Primary text: #FAFAFA
- Secondary text: #9E9E9E
- Tertiary text: #6B6B6B
- Border: #2A2A2A
```

**Accent (TE Orange):**
```
Light: #FF6B00
Dark: #FF8C42 (warmer for dark backgrounds)
```

**Usage rules:**
1. Monochromatic base, single accent
2. Accent for interactive elements only
3. Don't mix multiple accent colors
4. Color = function mapping
</color_system>
